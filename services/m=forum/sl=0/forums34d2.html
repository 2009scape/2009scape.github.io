A Brief on System Information (UPDATED 18 Oct 2009)

There is a significantly better tool available for Linux users, which is platform independent. This tool is known as "hardinfo", which is short for "hardware information". This tool does a very good job of pulling as much information as it can, and presents it in a very easy to read format.

------

To install hardinfo:

1) Go to your Terminal.
2) Type the command "sudo apt-get install hardinfo", without the quotes. Satisfy all security requirements, and wait for it to finish installing.
3) It's complete! Now either run it through the terminal "hardinfo", or through System -> Preferences -> System Profiler and Benchmark.

[In Karmic and above, the menu will be found under Applications -> System Tools.]

------

<Finding the vital information>

Most, if not all of the information required to diagnose your machine and its problems can be found directly in the "Summary" menu. To get to this menu:

1) Run hardinfo via Terminal, Alt+F2, or navigate to System -> Preferences -> System Profiler and Benchmark (Applications -> System Tools in Karmic and above).

2) Click on the Summary side menu. In the right hand side, in order, you will find your Processor (model name), Memory, OS, Resolution, OpenGL Renderer (which essentially translates to your graphics card), and your X11 Vendor.

If I, another maintainer, or someone else needs more information, we will not only request it, but we will tell you where to find it. I am confident though that hardinfo will be so simple to use, you probably won't need our advanced guiding with it.

Troubleshooting and Installing Java

There are two distinct ways to install Java - one is easier, but not always the best option; the other is harder, but keeps your Java up-to-date. Choose your method wisely.

---

[NOTE] OpenJDK will work fine with RS if you're using Firefox. Follow these steps if you want Sun/Oracle Java, or wish to use Java in a different browser.

The way to figure out which Java version you're running is by use of the 'java -version' command. This will display information in the following manner:

user@host:~$ java -version
java version "1.6.0_15"
Java(TM) SE Runtime Environment (build 1.6.0_15-b03)
Java HotSpot(TM) 64-Bit Server VM (build 14.1-b02, mixed mode)

---

Installing Java � The *easy* way

0) Add the "partner" repository in Lucid and above. Reference the release notes for Ubuntu 10.04 (Sun Java moved to the Partner Repository) for clearer instructions.

1) Run the Synaptic Package Manager. This is found in System -> Administration -> Synaptic Package Manager.

2) Satisfy the security requirements, and go to the search bar, located at the top of the program, or in Edit -> Search, or press Ctrl + F.

3) Type in 'sun-java6' and hit Enter. A list of packages matching that should appear.

4) Check the following package [all prefixed with sun-java6]: fonts, bin, jdk, jre, and plugin.

5) Apply the changes by hitting the green check at the top.

6) Wait for it to finish downloading and installing, then go to a terminal to confirm Java is installed by typing 'java -version'.

The distinct advantage of this is that it's virtually painless, and for most players, further work on Java's installation is not required. However, this does not always give you the latest Java.

[WARNING] Unless RuneScape does not function properly, or you want the newer Java releases, don't attempt the other method!

Troubleshooting and Installing Java

Installing Java � The *hard* way

[ You MUST have the sun-java6 packages installed for this to properly work. This also makes use of the Java Development Kit. ]

[WARNING] Any time a command requires the prefix �sudo�, it means that it is being done with super-user privileges. Please exercise caution whenever told to perform a sudo command. The following sudo commands are safe enough to be done without causing permanent damage to your machine, although, if done incorrectly, may cause damage to your Java installation.

1) Go to the Java website and download the latest packages. It *must* be the self-extracting, binary JDK, and not the JAR or RPM! Place this file to your desktop.

2) After the file has downloaded, type the following command: 'cd ~/Desktop; chmod +x jdk-<TAB>', where <TAB> is when you hit the tab button, which auto completes the file name.

3) Type the command .'/jdk-<TAB>', and agree to the license agreement.

4) You should now have a directory that has the binaries in it for Java.

5) Type the command �sudo mv jdk1-<TAB> /usr/lib/jvm�. This moves it to /usr/lib/jvm/, the global location of Java on Ubuntu. Then, type 'cd /usr/lib/jvm', which will put you in the /jvm directory.

6) Type the command 'sudo rm java-6-sun'. This will *remove* your symbolic link to Java on your system � please exercise caution at this point. If no symlink here exists, continue to the next step.

7) Type the command 'sudo ln -s jdk-<TAB> java-6-sun'. This will recreate your symbolic link to the new Java on your system.

8) Type the command 'sudo update-java-alternatives -s java-6-sun'. This command will recreate all previously broken links to your old JRE.

The distinct advantage of this is that it will always give you the latest Java, and for most players, that may actually be required. However, this is definitely not an easy thing to do.

Advanced Topics

- New Java Plugin [UPDATED 5 January 2010]

The new Java Plugin adds some speed and reliability performances to Java applets. It currently only has the 'server' run-mode implementation, so performance may seem a bit sluggish at first.

This is enabled by default; if you wish to revert to the old plugin (32-bit only), the guide to do that is at the bottom of the first page.

- Compositing Manager [UPDATED 27 March 2009]

Compositing managers allow for really nice eye candy with OpenGL and a graphics card that can support it. Recently, users of ATI cards would have conflicts with compositing and OpenGL applications (RuneScape uses JOGL, a port of OpenGL to Java), but with the ATI Catalyst 9.3 driver, it is possible to have these extra features and play RuneScape at the same time. NVidia cards have had a workaround for months, and could use these features alongside playing RuneScape.

If you do not wish to have this, you can disable compositing features through the following tool.

1) Go to a Terminal and type in the command "sudo apt-get install fusion-icon", without quotes. Satisfy security requirements, and wait for it to finish installing.

2) Run this command by pressing Alt+F2 and typing "fusion-icon" into the dialog. This will create the icon in your notification area.

3) *Right* click (left click will do nothing), and if you wish to switch rendering schemes, navigate to "Select Window Manager", and click "Metacity".

4) If you wish to start this by default at all times, navigate to System -> Preferences -> Sessions, click on the "Add" button, type in a simple name (like Fusion Icon), type the command "fusion-icon" into the 'command' box, and place any comment you like, but it's not advised to leave it blank. While your machine is logged in, it will be displayed in the notification area, and not get in your way.

Advanced Topics

- 64-bit OS + RuneScape [UPDATED 1 February 2010]

As of 1 Feb 2010, RuneScape HD will work properly within 64-bit Linux, including the new Java plugin. If you have any problems though, don't be afraid to post about it.

- Sound with Linux in RuneScape [UPDATED 28 November 2009]

<In Karmic, using Google Chrome with the following methods will work. This tactic will NOT work with Firefox.>

For an unknown reason, sound in Java will not work unless you're not listening to any other music at the moment. That's all good and dandy; just don't turn on your music until after RS has loaded, right? Wrong. Java will usurp control of the sound card, rendering it absolutely unusable by anything else (but not damaged) until you log out.

To eliminate this, simply wrap Firefox around PulseAudio. The way to do this is �padsp firefox� (or "aoss firefox", for those using ALSA - Thanks Redbloodx5!).

I don't like typing that every single time I load Firefox, though, so there is a simple way to do this.

1) Right-click on your menu bar, and select �Edit Menus�.
2) On the left-hand side, click on the �Internet� Menu. All of your Internet applications should pop up.
3) Navigate to your web browser (Firefox, Opera, etc), right-click on it, and select �Properties�.
4) That is simply a quick link to the program you're trying to run. Prepend (put at the front) the aforementioned command, hit Close, and you're done.

- Screen flickering [UPDATED 5 January 2010]

A unique problem occurred if you moused over the Jagex logo while playing a game; your game screen would go white. It has been a while since I've observed this behavior, so it is unlikely an issue now. If you *do* notice this, though, please don't hesitate to post.

Frequently Asked Questions

Q) Can RS run on Linux?

A) Short answer: Yes.
Long answer: RuneScape can run on pretty much any architecture that Java has been ported to, so this should cover your PC. There are, of course, hardware limitations, which would prevent RuneScape from running efficiently or at all, so at this time, there won't be any RS for your mobile or your PS3 through Java.

Q) Does Linux cost anything?

A) No. The majority of distributions are free; if you wish to pay, you'll probably be able to pay for support.

Q) Can Linux get viruses?

A) Not in the sense that you think. Every operating system can contract viruses, but the way that Linux handles executable files to begin with is unique from Windows. On most modern distros, you are never logged in full-time as the root account by default, so the damage done is limited to only your home directory. Next, if you were to download a malicious program, you would have to make it executable. After that, you would have to run it! That's a whole lot of work for someone to try to infect your machine, and an alert user would catch it before it was too late. Lastly, there are only a handful of viruses for Linux that not only can run without your permission, but can actually propagate themselves to other machines, so the odds of you actually getting one are incredibly slim.

Q) Sound is a few seconds behind a normal action! What's wrong?

A) The problem here is with the sound on Linux. To be frank, there are a lot of sound options on Linux platforms (OSS, ALSA, PulseAudio), and I'm not sure if there's a conflict or something someplace, but the sound from plugins such as Flash or Java come out slightly slower than normal. Your best option is to report the issue with Sun, developers of PulseAudio, or Adobe, as they are much more likely to come up with the solution. At this time, there's no readily available fix.

Terminology

"sudo, super-user, su, root" - Equivalent to "Administrator"; your account can be granted temporary root powers

"Terminal" - A simple environment for one to provide commands to the system

"Linux" - A free and open-source kernel, which is at the heart of many distributions

"Distro, distribution(s), flavor" - The variant of the Operating System that you're using, which runs the Linux kernel

How to Install Proprietary Drivers in Ubuntu
----

Proprietary drivers are drivers that do not have publicly available source code. This means that any and all misnomers, problems, bugs or errata have to be handled through the vendor, and not the open source community.

----

I. Installing the Easy Way

For most machines, driver installation can be done in a few simple steps.

1) Navigate to System > Administration > Hardware Drivers.

2) Click on the *most* current version you can see. If there are no "current" versions, simply click on the proprietary driver available.

3) Once it finishes installing, log out/restart X, and enjoy your new proprietary drivers.

II. Installing the (very!) Hard Way

(This method is suggested for non-Ubuntu derivatives as well)

1) Find out which family your card belongs to. To do this, navigate to Applications > System Tools > System Profiler and Benchmark, click on PCI devices, and select VGA compatible controller.

2) Navigate to the manufacturer's site (either AMD or NVIDIA), and download an appropriate driver for your machine.

3) Download the binary package, and save it either in /home or somewhere convenient in your /home directory.

4) Navigate to the Terminal, and navigate to where you saved your binary.

5) Run the command "chmod +x $name_of_binary", where $name_of_binary is the name of the binary you downloaded.

[NOTE] You will have to do the following parts *EVERY* time a new kernel comes out!

6) AMD users: You may run sudo ./$name_of_binary; after you do, reboot your machine, and you're set.

7) NVIDIA users: Save everything you're working on and press CTRL+ALT+F1. This will drop you to a shell. From there, log in as you would with the GUI, run the commands "sudo service gdm stop && sudo ./$name_of_binary", and say "yes" to anything the installer asks.

8) NVIDIA users: Run the command(s) "sudo service gdm start && exit". This will take you back to your desktop.
